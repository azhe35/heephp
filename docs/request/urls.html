# URL 模式

# URL 作为 MVC 模式的一部分，其决定了该从哪个控制器读取并返回数据。

# 以下为正确的路由示范

HeePHP 的 URL 非常灵活，使用也非常简单，通过url助手函数即可生成。

URL 的简单示例：

//多应用
//home应用下index控制器index方法
'/home/index/index'

//home应用下users控制器的index方法 。由于index为配置中的默认方法名，所以可以省略。
'/home/users'

//默认控制为index 默认方法名为index，因此该URL访问admin应用的index控制器的index方法。
'/admin'

//home应用的users控制器的edit方法 /1为edit方法的参数
'/home/users/edit/1'

//单应用
//index控制器index方法
'/index/index'

//下users控制器的add方法
'/users/add'

//users控制器的index方法 。由于index为配置中的默认方法名，所以可以省略。
'/users'

//默认控制为index 默认方法名为index，因此该URL访问index控制器的index方法。
'/'

//users控制器的edit方法 /1为edit方法的参数
'/users/edit/1'

URL 顺序 /应用/控制器/方法名/参数1的值/参数2的值/参数n的值
参数值的顺序须按照方法定义的参数顺序，（如需更改参数顺序，请参照路由章节配置路由）
