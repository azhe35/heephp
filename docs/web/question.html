    # 常见问题

    # 区别

    HeePHP的理念是简洁、快速，因此有一些功能是区别于Laravel、Yii2、ThinkPHP，实际上吸纳了其他框架的优点，同时又有自己的特色。

    # URL模式

    URL的模式，HeePHP仅支持通过应用\控制器\方法\参数值1\参数值2访问，且参数必须按照方法中定义顺序，不能带参数名。同时可以通过PARMS常量数组，获取参数值列。注意：不包括参数名。

    # URL生成

    url函数的使用：url('test',['id'=>1,'name'=>'apple']) 等同于 url('test',[1,'apple'])
    生成的URL：控制器\test\1\apple

    # 配置文件

    配置文件包括：总配置文件app目录config.php,不同应用中的配置文件app\home,app\admin....等目录的config.php,应用中的配置项会覆盖总配置文件项。
    配置项不支持运行时修改。

    # request请求 post、get

    request函数用于读取或设置post、get、session、server中的值，在使用post、get时需注意：对于提交的html、引号会进行转义，因此对于得到的数据,如果有HTML或者引号等字符，需要进行转义：stripslashes(html_entity_decode(request('post.item')))
    对于提交的SQL关键词会过滤：Select Update Insert Delete form 等

    # 分页

    使用ORM的page()方法，可以获取分页信息以及分页的html代码，分页的URL参数默认为page_页码,可以通过pageparm()方法设置参数名。
    需要说明：page参数仅能在URL中参数值列表之后 例如：应用\控制器\方法\参数值1\参数值2\page_1
